# Decision journal — 25


## Decision
Complete project investigation for issue #25. Key findings:

1. **Canonical provider-id list lives in `apps/web/src/lib/providers.ts`** as `DEFAULT_PROVIDERS` array (26 providers). The provider IDs are stable kebab-case identifiers: `aws-polly`, `azure-speech`, `gcp-tts`, `elevenlabs`, `openai`, `qwen3-tts`, `gemini-tts`, `deepgram`, `cartesia`, `lmnt`, `edge-tts`, `kokoro`, `dia`, `bark`, `kittentts`, `marvis`, `vibevoice`, `orpheus`, `soprano`, `pocket-tts`, `spark-tts`, `chatterbox`, `chatterbox-turbo`, `outetts`, `voxcpm`, `huggingface`. The SDK `ProviderId` union type in `packages/sdk/src/types.ts` only covers 9 cloud providers and needs updating.

2. **`/data/voices.json` is the source of truth** — currently has NO version stamp (`{ "voices": [...] }`). It can be generated from DynamoDB via `publish-catalog.mjs --from-db` or hand-maintained locally.

3. **`/api/v1/providers.json` and `/api/v1/voices.json` already carry `generatedAt` + `total`**. The issue comment confirms this satisfies the "version stamp on catalog artifact" part of the request.

4. **Both static and API voices derive from the same source**: `/data/voices.json` → `publish-catalog.mjs` → static `/api/v1/voices.json`; AND `/data/voices.json` → `infra/build.sh` copy → synthesis-router Lambda → `/v1/voices` endpoint. They are reconciled by construction.

5. **No CHANGELOG.md exists** in the repo.

6. **Remaining work** (per issue comment): (a) Document provider-id stability/deprecation policy (highest priority), (b) Optionally add content hash alongside generatedAt, (c) Backfill version stamp on `/data/voices.json` itself.

7. **`packages/shared/` is an empty placeholder** — a `CatalogVersion` type could be defined there but there's essentially nothing to build on; defining it in `apps/web/src/lib/types.ts` or `packages/sdk/src/types.ts` would be more practical.
